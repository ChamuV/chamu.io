Overview

When you watch a flock of birds in the sky, it‚Äôs hard not to miss the dynamics: they bunch together, fan out, change direction at the same moment, and sometimes split before regrouping again. It looks organic and slightly chaotic, yet the underlying behaviour can be modelled surprisingly well using simple rules from dynamical systems and models often associated with ‚Äúchaos theory‚Äù. Once I learned this, I wanted to try implementing it myself.

I also wanted a background for my website that felt a bit different‚Äîsomething that reflected my interest in mathematics while staying only mildly distracting. A lightweight Boids simulation turned out to be a good solution: a calm flock drifting behind the content, driven by simple rules but producing surprisingly rich behaviour.

How It Works

The Boids model, introduced by Craig Reynolds, recreates flocking behaviour using only three simple rules applied independently to each agent. Each boid looks at its nearby neighbours and updates its velocity based on:

  • Separation ‚Äì steer away from neighbours that are too close to avoid collisions.
  • Alignment ‚Äì adjust direction to match the average heading of nearby boids.
  • Cohesion ‚Äì steer towards the average position (centre of mass) of the local flock.

These rules are combined at each animation step, and the resulting velocity advances the boid forward. Although each rule is simple and entirely local, their interaction produces the familiar emergent patterns of flocking‚Äîgroups forming, splitting, turning, and reuniting.

Implementation Details

The implementation is intentionally lightweight so it can run smoothly in the background without burning the laptop:

  • Written in vanilla JavaScript and rendered on an HTML5 <canvas> using the 2D drawing context.
  • Uses requestAnimationFrame for smooth, browser-synchronised updates.
  • Keeps the number of boids modest so a simple all-pairs neighbour search is sufficient for good performance.
  • Boids wrap around screen edges so the flock never disappears.
  • Colours and line thickness are chosen to stay in the background and not compete with the content of the page.

The same script is reused across the site: any page with data-bg="boids" and a <canvas id="boids-bg"> element will automatically get the flocking background.

This Boids engine is designed to be easy to tweak. Future extensions I‚Äôm considering include:

Because the code is small and self-contained, it also serves as a fun sandbox for experimenting with ideas from dynamical systems and collective behaviour.
