Overview

During my gap year after high school ‚Äî right in the middle of the COVID lockdown ‚Äî I suddenly had a lot more time at home. I came across a series of DIY robotics buggy videos online and decided to try building my own using an old Raspberry Pi left over from a physics project.

What started as a small experiment grew into a three-month build involving design, wiring, motor control, and software. By the end, I had a Raspberry-Pi-powered buggy that could connect to my phone, follow live commands, avoid obstacles, and stream a camera feed for remote navigation. It was the first time I properly combined hardware and software in a single project, and it became a solid introduction to robotics and embedded systems.

Description

The buggy was built following the Raspberry Pi Foundation‚Äôs Build a Buggy guide. The core idea was simple: assemble a small two-wheel robot platform, wire the motors to a motor controller board, and then program the movement using Python.

The basic version required a small set of components:

  • Raspberry Pi with GPIO header
  • Motor controller board
  • 2√ó DC motors with wheels
  • Ball caster for stability
  • 4√ó AA battery pack
  • Jumper wires and tape

Once the wiring was complete, I programmed the buggy to move using Python: forward, backward, turning, and running simple movement sequences. Most of the hardware control was done using the gpiozero library, which greatly simplifies GPIO access.

After the basic buggy worked, I added two line-following infrared sensors and experimented with an HC-SR04 ultrasonic ‚Äúsonar‚Äù distance sensor. For the ultrasonic sensor, I used Python‚Äôs time module to measure echo return durations and the signal module to handle timeouts safely.

I later discovered that I could pair the buggy with an Android app called Big Dot that sends continuous direction commands over Wi-Fi. Combining this with the sonar sensor allowed me to create a simple follow-me system: the phone kept the buggy moving toward me, and the ultrasonic sensor stopped or redirected it when obstacles were detected.

This produced a basic ‚Äúfollow-me while avoiding obstacles‚Äù behaviour, built entirely from inexpensive components and a few Python scripts.
